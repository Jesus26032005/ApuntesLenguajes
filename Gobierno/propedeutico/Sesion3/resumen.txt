**Comentarios en Java: tipos y usos**  
- Se definen los comentarios como texto para el programador que no se ejecuta ni genera errores.  
- Dos tipos principales:  
  - Comentarios de una sola línea: iniciados con `//`. Todo lo que sigue en esa línea es ignorado por el compilador.  
  - Comentarios de varias líneas: iniciados con `/*` y terminados con `*/`. Pueden abarcar tantas líneas como se desee.  
- Los comentarios se pueden poner en cualquier parte del código para documentar su funcionamiento y facilitar su comprensión.  
- Se menciona un tercer tipo, los comentarios de documentación (JavaDoc), que no se abordarán en este curso básico.

**Identificadores en Java: definición y reglas para nombrarlos**  
- Un identificador es el nombre dado a una variable, clase o método.  
- Reglas para nombrar identificadores:  
  - Deben iniciar con una letra (mayúscula o minúscula) o con los símbolos permitidos: guion bajo `_` o signo de pesos `$`.  
  - No pueden iniciar con números ni otros símbolos como `#`, `@`, `=`, `+`.  
  - Son sensibles a mayúsculas y minúsculas, por ejemplo, `Manuel` y `manuel` son identificadores distintos.  
  - No tienen límite en la longitud, aunque se recomienda mantenerlos razonablemente cortos por facilidad de uso.  
- Buenas prácticas: usar el estilo "camelCase" para identificadores compuestos, empezando con minúscula y cada palabra siguiente con mayúscula (ejemplo: `tecnológicoNacionalDeMéxico`).  
- Se recuerda que ciertos términos son palabras reservadas de Java y no pueden usarse como identificadores (ejemplo: `do`, `for`).

**Variables en Java: concepto y tipos primitivos**  
- Variables son "cajitas" donde se almacenan valores.
- Java es un lenguaje fuertemente tipado, por lo que cada variable debe ser declarada con un tipo específico.  
- Tipos primitivos proporcionados por Java (sin importar bibliotecas externas):  
  - **Booleanos:** solo pueden tener valores `true` o `false`.  
  - **Caracteres (`char`):** permiten almacenar un solo carácter (letra, número o símbolo). Se escriben encerrados en comillas simples `'A'`.  
  - **Numéricos enteros:**  
    - `byte`: números entre -128 y 127 (8 bits).  
    - `short`: números más grandes que `byte`.  
    - `int`: enteros más grandes que `short`.  
    - `long`: enteros más grandes que `int`.  
  - **Numéricos con decimales:**  
    - `float`: números decimales más pequeños, requiere sufijo `F` para asignación (ejemplo: `3.141F`).  
    - `double`: números decimales más grandes, es el tipo decimal por defecto.  

[00:56:30]  
**Detalles sobre variables tipo byte y desbordamiento (overflow)**  
- `byte` almacena valores entre 0 y 255 (*Nota:* en realidad es -128 a 127, pero el instructor menciona 0 a 255, posible simplificación).  
- Si se suma un valor fuera del rango, ocurre un desbordamiento y el valor se "recicla" comenzando desde el límite inferior (overflow).  
- Ejemplo: 255 + 1 vuelve a 0.  

[01:01:03]  
**Asignación entre tipos numéricos: compatibilidad y errores**  
- Se puede asignar un valor pequeño a una variable de tipo más grande sin problema (ejemplo: `byte` a `int`).  
- No se puede asignar un valor grande a una variable de tipo más pequeño sin conversión explícita, esto genera error de compilación.  

[01:02:31]  
**Variables de tipo flotante y doble: asignación y precisión**  
- Al asignar un número decimal literal, Java lo interpreta por defecto como `double`.  
- Para asignar a un `float` se debe usar el sufijo `F` (ejemplo: `3.141F`).  
- Se puede asignar un `float` a un `double` sin problema, pero no al revés sin conversión explícita (cast), ya que puede perder precisión.  
- El cast obliga al programador a aceptar la posible pérdida de información.

[01:26:23]  
**Diferencia entre `byte` y `char`**  
- `byte` almacena un valor numérico entre 0 y 255 (o -128 a 127).  
- `char` almacena un carácter representado internamente por un valor numérico según la tabla ASCII.  
- Ejemplo: `'A'` tiene valor ASCII 65, `'a'` tiene 97.  
- Se puede operar con valores numéricos de tipo `byte` para incrementar o sumar.

- `double` soporta números con más precisión y tamaño que `float`.  
- Al asignar un número decimal sin sufijo, Java lo interpreta como `double`.  
- Para asignar a `float` es necesario usar `F` al final del literal.  
- Se explica el riesgo de pérdida de datos si se fuerza la conversión de `double` a `float` (cast).

### Preguntas frecuentes (FAQ)

- **¿Qué es un identificador en Java?**  
  Es el nombre que se da a una variable, clase o método, siguiendo reglas específicas para su validez.

- **¿Por qué los comentarios son importantes?**  
  Permiten documentar el código para facilitar su comprensión y mantenimiento sin afectar la ejecución del programa.

- **¿Se pueden usar números al inicio de un identificador?**  
  No, deben comenzar con letra, guion bajo o signo de pesos.

- **¿Qué pasa si asigno un valor fuera del rango a un `byte`?**  
  Ocurre un desbordamiento (overflow), y el valor se reinicia desde el límite inferior.

- **¿Cómo se crean objetos en Java?**  
  Usando el operador `new` y el constructor de la clase correspondiente.

- **¿Cuál es la diferencia entre clase y objeto?**  
  La clase es la plantilla general, el objeto es una instancia específica con valores concretos.
